# Changelog

## [2.7.0] - 2026-03-06

### 新增
- **内容重复度分析**（框架分析板块）：对主播逐字稿进行话题语义聚类，识别反复提及的内容模块，输出频次表格并标注重复类型（主动强调/无意识重复/口头禅）
- **直播标题质量评估**（框架分析板块）：5 维度打分（关键词相关性、吸引力、简洁度、人群契合度、差异化），总分 100 分，并提供 3 个优化版标题
- **评论互动深度分析**（评分板块）：重构原「互动回应质量」为 4 个子模块
  - 互动率量化：有效评论数 ÷ 平均在线人数，对标行业基准 ≥20%
  - 评论回复对照表：逐条评论 vs 逐字稿对照，标注回复状态和时效
  - 评论回复率汇总表：6 项关键指标
  - 未回复重要评论清单 + 建议回复话术
- **直播间元数据传入 LLM**：`_buildDataContext` 新增直播标题和主播名称，供标题评分等分析使用

## [2.6.0] - 2026-03-05

### 新增
- **合规分析基金/投顾双模式**：一键生成报告时弹出直播类型选择（基金直播/投顾直播），不同类型使用完全不同的合规审核提示词
  - 投顾版：证券类核心红线（荐个股/K线/承诺收益/诱导跟单/风险提示）
  - 基金版：基金销售合规红线（保本承诺/夸大业绩/诱导申购/适当性/法定风险提示）+ 信息披露
  - 记住用户上次选择，下次自动高亮
- **直播质量评分**（第4板块）：5维度 × 20分制量化评分
  - 互动活跃度、内容节奏与结构、话术质量、观众留存表现、互动回应质量
  - 输出总评分表格 + 逐维度详细分析
- **整体优化建议汇总**（第5部分）：前4板块并行完成后自动触发
  - 综合4个板块提取所有优化点，按 P0-P3 优先级排序
  - 输出优化清单表格 + Top 3 关键行动方案 + 直播总评
- **直播类型选择弹窗 UI**：勃艮第红 Warm Luxury 风格，支持高亮上次选择

### 优化
- `buildInput` 重构为支持 `liveType` 和 `previousResults` 参数
- `CONTEXT` 拆分为 `CONTEXT_BASE` + `LIVE_TYPE_CONTEXT`，动态拼接直播类型背景
- 并行生成流程升级：选择类型 → 4板块并行 → 自动汇总（共5部分）
- HTML/TXT 报告包含全部5个分析部分

## [2.5.0] - 2026-03-05

### 新增
- **合规性分析 prompt 全面重写**：基于证监会规则和抖音平台财经类直播规范调研，从 2 个维度扩展至 5 大维度
  - 证券类核心红线（荐个股/K线讲解/确定性承诺/诱导跟单/风险提示）
  - 引流与私域导流（微信/外部链接/群组）
  - 通用话术合规（极限词/虚假宣传/虚假限时/敏感领域）
  - 主播形象与信息披露（虚假人设/姓名编号公司展示）
  - 观众互动合规（个股提问应对）
- 输出增加整体合规评级、合规亮点、改进建议小结

### 修复
- HTML 报告标题与下载文件名统一

## [2.4.0] - 2026-03-04

### 新增
- **三板块并行分析**：一键生成时三个板块同时发起 AI 请求，速度提升约 3 倍
- **Service Worker 保活**：页面打开时自动建立心跳连接（25s 间隔），防止 Chrome 休眠中断 LLM 请求
- **连接断开自动重试**：意外断连最多重试 3 次（间隔递增 1s/2s/3s），重试过程实时提示
- **错误反馈 Toast**：醒目的错误提示（SVG 图标 + 关闭按钮 + 8 秒自动消失 + 左侧红色边条）
- **意外断连检测**：修复 Service Worker 休眠时错误被静默吞掉的 bug

### 优化
- **合规性分析 prompt**：区分持证合规与话术违规，新增证券类直播专项检查（确定性承诺、诱导跟单等），增加整体合规评级
- **停止生成按钮**：改为与采集区一致的浅红色风格
- **LLM 多连接架构**：`_activePort` 重构为 `_activePorts` Map，`stop()` 支持按 sectionKey 停止单个或全部
- 所有 UI 图标统一为 SVG，移除 emoji

## [2.3.0] - 2026-03-04

- 添加 `callbackFired` 锁防止回调重复触发

## [2.2.0] - 2026-03-04

- **停止按钮配色**：统一「停止生成」按钮为与「停止」按钮一致的勃艮第红配色
